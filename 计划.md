# macOS 摄像头自拍软件开发计划

## Context

用户需要一个 MacBook 自拍软件，能调用系统摄像头并支持 macOS 原生的**人物居中 (Center Stage)** 等全部系统视频效果。需要照片浏览管理功能。个人本地使用，无需签名和沙盒化。

## 技术方案

| 维度 | 选择 |
|---|---|
| 语言 | Swift 5.9+ |
| UI 框架 | SwiftUI + NSViewRepresentable（桥接摄像头预览层） |
| 摄像头框架 | AVFoundation |
| 最低部署目标 | macOS 13.0（Center Stage 要求 macOS 12+，macOS 13 更稳定） |
| 照片存储 | 本地文件系统（~/Pictures/Mac拍照软件/） |
| 分发方式 | 本地运行，无需代码签名 |
| 构建方式 | Xcode 项目（需要 Info.plist 声明权限） |

## 项目结构

```
Mac拍照软件/
├── Mac拍照软件/
│   ├── Mac拍照软件App.swift           # @main 入口
│   ├── ContentView.swift              # 主界面布局
│   ├── CameraPreviewView.swift        # NSViewRepresentable 桥接预览层
│   ├── CameraManager.swift            # 核心：AVCaptureSession + 设备管理 + 视频效果
│   ├── PhotoManager.swift             # 照片存储和管理
│   ├── PhotoGalleryView.swift         # 照片浏览界面
│   ├── PhotoItem.swift                # 照片数据模型
│   └── Info.plist                      # NSCameraUsageDescription 权限声明
```

## 实施步骤

### 第 1 步：创建项目结构和 Info.plist
- 创建 Xcode 项目（macOS App，SwiftUI 生命周期）
- 创建 Swift 源文件目录结构
- 编写 Info.plist，添加 `NSCameraUsageDescription` 权限描述
- 配置部署目标为 macOS 13.0

### 第 2 步：实现 CameraManager（核心）
**文件：`CameraManager.swift`**

- `ObservableObject` 管理所有状态
- **AVCaptureSession 配置**：
  - `DiscoverySession` 发现可用摄像头
  - `AVCapturePhotoOutput` 拍照
  - 后台线程运行 session（`startRunning()` 是阻塞调用）
- **系统视频效果集成**（全部支持）：
  - **Center Stage（人物居中）**：
    - `AVCaptureDevice.isCenterStageSupported` 检查
    - `centerStageControlMode = .cooperative`
    - 选择支持 Center Stage 的格式
  - **人像模式（Portrait Effect）**：
    - `AVCaptureDevice.isPortraitEffectSupported` 检查
    - `isPortraitEffectEnabled` 开关
  - **摄影室灯光（Studio Light）**：
    - `AVCaptureDevice.isStudioLightSupported` 检查
    - `isStudioLightEnabled` 开关
  - **反应效果（Reactions）**：
    - `AVCaptureDevice.isReactionEffectSupported` 检查
    - `isReactionEffectEnabled` 开关
  - 提供统一的 `showSystemUI(.videoEffects)` 入口
  - **互斥逻辑**：Center Stage 和人像模式不能同时开启，切换时自动关闭对方
- **摄像头切换**：前置/后置切换
- **拍照**：`AVCapturePhotoCaptureDelegate` 回调

### 第 3 步：实现 CameraPreviewView（预览桥接）
**文件：`CameraPreviewView.swift`**

- `NSViewRepresentable` 包装 `AVCaptureVideoPreviewLayer`
- `videoGravity = .resizeAspectFill`
- 自拍镜像：`isVideoMirrored = true`
- 响应窗口 resize

### 第 4 步：实现 PhotoManager（照片管理）
**文件：`PhotoManager.swift`**

- 照片保存到 `~/Pictures/Mac拍照软件/` 目录
- **首次使用时自动创建目录**（`FileManager.createDirectory`）
- 照片元数据管理（时间戳）
- 照片列表加载和排序
- 删除照片功能

### 第 5 步：实现 ContentView（主界面）
**文件：`ContentView.swift`**

- **预览区域**：占主要面积，实时摄像头画面
- **控制面板**（底部工具栏）：
  - 拍照按钮（大圆形，居中）
  - Center Stage 开关
  - 人像模式开关（与 Center Stage 互斥提示）
  - 摄像室灯光开关
  - 摄像头切换按钮
  - 系统视频效果入口按钮
  - 照片浏览按钮
- **照片缩略图**：最近拍摄的照片预览

### 第 6 步：实现 PhotoGalleryView（照片浏览）
**文件：`PhotoGalleryView.swift`**

- 网格/列表展示所有拍摄的照片
- 点击放大查看
- 左右滑动/箭头切换
- 删除功能
- 导出/分享功能

### 第 7 步：权限和错误处理
- 首次启动请求摄像头权限
- 权限被拒绝时显示引导（打开系统偏好设置）
- 设备不可用时的友好提示

## 系统视频效果关键实现

```swift
// Center Stage（开启时自动关闭人像模式）
if AVCaptureDevice.isCenterStageSupported {
    AVCaptureDevice.centerStageControlMode = .cooperative
    AVCaptureDevice.centerStageEnabled = true
    // 互斥：关闭人像模式
    if device.isPortraitEffectSupported {
        device.isPortraitEffectEnabled = false
    }
}

// 人像模式（开启时自动关闭 Center Stage）
if device.isPortraitEffectSupported {
    device.isPortraitEffectEnabled = true
    // 互斥：关闭 Center Stage
    AVCaptureDevice.centerStageEnabled = false
}

// 摄影室灯光
if device.isStudioLightSupported {
    device.isStudioLightEnabled = true
}

// 打开系统视频效果面板
AVCaptureDevice.showSystemUserInterface(.videoEffects)
```

## 验证方式

1. 编译运行 → verify: Xcode 无错误，Cmd+R 成功启动
2. 摄像头预览 → verify: 实时画面显示，镜像正确
3. 拍照 → verify: 点击按钮后照片出现在 ~/Pictures/Mac拍照软件/
4. Center Stage → verify: 开关切换后画面自动跟踪人物
5. 人像模式 → verify: 开启后背景虚化
6. 互斥验证 → verify: 开启 A 时 B 自动关闭，反之亦然
7. 摄影室灯光 → verify: 开关后面部光线有变化
8. 摄像头切换 → verify: 前后摄像头画面正确切换
9. 照片浏览 → verify: 能查看、放大、删除已拍摄的照片
10. 权限处理 → verify: 首次运行弹出权限请求，拒绝后有引导提示
